# 使用者手冊 (Manual)

## 1. 這個外掛可以做什麼

- 在 Markdown 筆記裡用固定語法建立閃卡。
- 自動掃描目前檔案或整個 Vault，建立閃卡索引。
- 依 FSRS 排程進行複習（Again / Hard / Good / Easy）。
- 提供答案高亮與語法隱藏顯示，讓編輯與閱讀更清楚。

## 2. 快速開始

1. 在任一筆記輸入閃卡語法（見下一節）。
2. 執行命令 `掃描目前文件中的閃卡`。
3. 執行命令 `開始閃卡複習`，開啟複習視窗。

你也可以使用左側 Ribbon 快捷按鈕進行掃描、重建索引、開始複習與查看統計。

## 3. 閃卡語法

- `概念 :: 正向答案`：正向問答。
- `解釋 ;; 概念`：反向問答。
- `單字 ::: 翻譯`：雙向卡。
- `這是一個 ==填空== 範例`：挖空卡（cloze）。
- 多行/清單答案：
  - 第一行輸入 `問題 ::`
  - 下一行開始以縮排內容作為答案（Tab 或 4 個空白）

### 語法規格草案（待實作）
- 為了在連續文字中精準框選閃卡範圍，規劃新增 `fc` 包裹語法。
- 單行卡（規劃）：
  - `{{fc:: 正面 :: 背面 /fc}}`
- 多行/清單卡（規劃）：
  - 起始：`{{fc::`
  - 內容：沿用既有 `正面 ::` + 縮排答案規則
  - 結尾：`/fc}}`
- 設計理由：
  - 單行與多行都用 `/fc}}` 結尾，規則一致、好記憶。
  - 比只用 `}}` 更不容易在多行內容中誤判提早結束。
  - 起始必須是 `{{fc::`，可降低與一般文字、模板片段混淆。
- 解析保護規則（規劃）：
  - `code block`、`inline code`、`math` 區塊內不解析 `fc` 包裹語法。
  - 多行卡建議以獨立一行 `/fc}}` 作為結束標記。
  - 第一階段會保留舊語法相容，避免既有筆記立即失效。

### `fc` 語法快速輸入方式（建議）
- 方式 1：使用模板片段快速插入
  - 先準備固定模板：`{{fc:: 正面 :: 背面 /fc}}`
  - 透過 Obsidian 模板指令或熱鍵一鍵插入，減少重複手打。
- 方式 2：選取文字後一鍵包裝（規劃中）
  - 先選取一段文字，再用外掛命令自動包成 `{{fc:: ... /fc}}`。
  - 單行與多行都可由命令判斷並套用對應格式。
- 方式 3：文字縮寫自動展開
  - 在輸入工具設定縮寫（例如 `;fc`），自動展開成 `{{fc::  ::  /fc}}`。
  - 適合大量建立卡片時快速連續輸入。

## 4. 編輯器顯示行為

- 游標離開答案行時，`:: / ;; / :::` 語法符號會自動隱藏。
- 游標回到該行時，原始語法會顯示，方便直接編輯。
- 單行答案與 cloze 目前使用膠囊式高亮（chip）。
- 多行/清單答案使用獨立渲染策略，不跟單行答案共用 chip。
- 僅在點擊答案高亮區塊時才會進入該卡片的編輯顯示脈絡（顯示語法與對應 Block ID）；一般點擊行內非高亮區塊不會觸發。
- `[[target|alias]]` 在 chip 內會顯示 `alias`，不是原始 wikilink 字串。
- 一般 markdown link 在 chip 內會顯示連結文字。

## 5. 複習功能

- 複習按鈕：`Again`、`Hard`、`Good`、`Easy`。
- 評分結果會更新卡片狀態與下次到期時間。
- 可使用命令 `顯示閃卡統計` 查看總數、到期數與各學習狀態分布。

## 6. 設定頁

- 設定頁採分頁介面，包含：
  - `一般`
  - `答案高亮`
  - `AI`
  - `維護工具`
- `一般`分頁：
  - `資料目錄`：調整閃卡資料儲存路徑（預設 `_Flashcards`）。
  - `自動同步 Markdown 變更`：控制檔案變更時是否自動同步索引。
- `答案高亮`分頁：
  - `答案高亮設定`：可多選 `填空`、`單行答案`、`多行答案`、`雙向卡`。
  - `多行答案渲染樣式`：可選 `淡色背景帶`、`右側線條`。
  其中 `淡色背景帶` 會把整個多行答案區塊視為同一塊背景區域，而不是每行分開一段。
  其中 `右側線條` 會以整個多行答案區塊最右側的一條共用導引線顯示，不會每行各自畫一條。
- `答案高亮主題`（位於 `答案高亮` 分頁）：
  - `主題色`：控制高亮主色。
  - `透明度`：控制高亮整體透明度（預設低對比深色半透明）。
  - `啟用各樣式獨立顏色`：可展開各樣式的獨立顏色欄位，保留進階自訂空間。
- `AI`分頁：
  - `AI 設定（預留）`：目前僅為設定入口，實際 AI enrich 流程尚未正式接入。
- `維護工具`分頁：
  - `移除所有閃卡 Block ID`：清理 Vault 內 `^fc-xxxxxx` 並同步刪除對應索引。

## 7. 常用命令

- `掃描目前文件中的閃卡`
- `重建整個 Vault 的閃卡索引`
- `開始閃卡複習`
- `顯示閃卡統計`
- `移除 Vault 內所有閃卡 Block ID`
- `顯示答案高亮診斷`（開發/排查用途）

## 8. 常見問題

- 為什麼會看到 `^fc-xxxxxx`：
  - 這是卡片對應的 Block ID。一般情況會自動隱藏，編輯脈絡下才可能顯示。
- 為什麼剛輸入語法還沒進入複習：
  - 需要先執行掃描命令，讓卡片被索引。
- AI 功能現在能不能直接產生內容：
  - 目前是設定預留階段，完整 AI enrich 流程將在後續版本逐步開放。
