### 1. 語法體系

- `概念 :: 正向答案`：看到左邊考右邊。
- `解釋 ;; 概念`：看到左邊考右邊（反向思維）。
- `單字 ::: 翻譯`：雙向互考。
- `這是一個 ==填空== 範例`：自動轉換為挖空卡片。
- `==填空==` 顯示樣式：使用低對比淺灰膠囊，不再使用刺眼黃底高亮（V0.1.10）。
- **語法自動隱藏**：在編輯模式中，`::`、`;;`、`:::`、`問題 ::` 會在游標離開該行後自動隱藏（V0.1.12）。
- **編輯模式高亮修正**：已壓過 Obsidian 原生 `== ==` 黃底，並將答案高亮改為更平直的塊狀樣式，避免膠囊/毛毛蟲邊框感（V0.1.13）。
- **高亮覆蓋補強**：若 `==填空==` 位於答案高亮範圍內，內層 `cm-highlight / cm-formatting-highlight` 會統一吃外掛底色，不再露出原生黃底（V0.1.14）。
- **多行/清單模式**：`問題 ::` 後方換行並縮排（Tab/4格），下方所有內容皆視為答案。

### 2. AI 互動與豐富化

- **手動按鈕**：每張卡片行末顯示 Notion 風格的小圖示，點擊後觸發 Gemini 3 Flash 生成。
- **多模態功能**：支援圖片右鍵「AI 轉化為閃卡」，自動識別 OCR。
- **暗色模式友善**：優先要求 AI 生成 SVG 或透明背景 PNG。
- **錯誤處理**：失敗時提供「換模型重試」或「關閉」選項。

### 3. 複習與 SRS

- **四鍵評分**：Again (淡紅), Hard (淡橘), Good (淡綠), Easy (淡藍)。
- **FSRS 演算法**：進度存於 `_Flashcards` 儲存層（`data.json` manifest + `Cards/*.json`），不弄髒筆記。
- **遺忘預警**：過期太久的卡片，行末圖示變為「琥珀色」。

### 4. 設定頁與維護工具（V0.1.9）

- **資料目錄設定**：可在設定頁調整 flashcard 資料儲存路徑（預設 `_Flashcards`）。
- **分片儲存結構**：`data.json` 僅保存版本與卡片 ID 清單；每張卡片內容獨立保存於 `Cards/<blockId>.json`，更新單張卡時不需重寫整包資料。
- **自動同步開關**：可控制 Markdown 修改時是否自動同步索引。
- **答案高亮範圍**：可在設定頁以多選方式勾選 `填空`、`單行答案`、`多行答案`、`雙向卡`（V0.1.11）。
- **答案高亮主題**：可在設定頁自訂高亮底色與透明度，並可一鍵重設預設值（V0.1.12）。
- **高亮衝突修正**：已修正 Obsidian `== ==` 原生高亮覆蓋答案高亮的問題，編輯模式視覺層一致（V0.1.12）。
- **AI 設定預留欄位**：已提供 `aiEnabled`、Provider、Model、API Key 的設定入口，供後續 AI enrich 流程接入。
- **Block ID 清理工具**：可一鍵移除 Vault 內所有 `^fc-xxxxxx`，並同步清除對應卡片索引，讓筆記回到純 Markdown 狀態。
- **Block ID 即時附加**：當游標離開閃卡行時，會在下一個事件迴圈立即補上 Block ID（V0.1.10），不再有明顯 500ms 延遲。

### 5. 間隔重複 (Spaced Repetition) 核心設計邏輯

本軟體採用先進的 **FSRS (Free Spaced Repetition Scheduler)** 演算法，捨棄了傳統死板的「固定天數倍增（如 1、3、7 天）」法則，改採動態追蹤你的大腦記憶狀態。

#### 核心記憶三維度
FSRS 會為您的每一張閃卡計算以下三個維度：
1. **Difficulty (難度)**：動態評估這張卡片對您個人的天生難易度。
2. **Stability (穩定度)**：以「天」為單位，代表在記憶機率掉到 90% 之前，這張卡片能在你腦中維持多久。
3. **Retrievability (可回憶度)**：目前這個當下，您能成功回憶起答案的機率（會隨著時間流逝而指數下降）。

#### 按鈕與時間間隔的關聯（以「新卡片」首次複習為例）
每次翻開閃卡背面後，請根據您「回想時的費力程度」點擊按鈕，FSRS 將會如此反應：

- **Again (忘記)**：完全沒印象或答錯。
  - **發生什麼事**：卡片進入「學習中 (Learning)」狀態。記憶穩定度歸零。
  - **下次複習**：通常在 **1 分鐘內** 或非常短的時間內會再考你一次，直到你記起來。
- **Hard (困難)**：想了很久或差點答錯。
  - **發生什麼事**：系統判定記憶尚未固化，增加這張卡片的難度參數。
  - **下次複習**：大約 **5 到 10 分鐘**，或數小時內就會再次出現。
- **Good (良好)**：順利回想起答案（這是最標準的情況）。
  - **發生什麼事**：系統判定記憶成功，開始進入正常的遺忘曲線排程。
  - **下次複習**：大約在 **1 天後**。
- **Easy (簡單)**：秒答，覺得單字或概念太過簡單。
  - **發生什麼事**：系統大幅提升穩定度並降低難度。
  - **下次複習**：大約在 **3 到 4 天後** 才會再次出現。

#### 複習次數與動態調節
隨著複習次數（Reps）增加，如果你連續選擇 Good / Easy，複習的間隔會漂亮地拉開（例如：1天 → 4天 → 12天 → 1個月）。但如果你在某次複習突然忘記按了 Again（產生 Lapse 遺忘發生），系統會給予「遺忘懲罰」，將下一次的間隔時間大幅縮短，確保您能重新建立好大腦的突觸連結。

> **核心精神**：請對自己誠實。不需要去猜測「我現在按 Good 明天會不會出現」，只要根據當下回想的難度自然點擊即可，FSRS 演算法會自動為您尋找大腦最有效率的複習黃金時間點。
