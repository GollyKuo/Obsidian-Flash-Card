# V0.1.23 重做步驟

1. 建立乾淨起點  
`git restore` 到 `0.1.22`，確認工作樹乾淨，並移除衝突殘留檔。

2. 鎖定唯一語法規格  
先寫死只支援最後確認的內嵌語法，不再雙規格並存，並列出合法/非法範例。

3. 先寫測試（Red）  
先補 parser 測試與高亮測試，不改主程式，先讓測試故意失敗。

4. 實作 parser（Green-1）  
只改解析層，讓單行卡與填充卡都能正確辨識；非法語法一律不解析、不 fallback。

5. 實作高亮規則（Green-2）  
只高亮答案區。填充卡只高亮 `==答案==`，不高亮整段包裹。

6. 實作語法隱藏規則（Green-3）  
閱讀模式隱藏語法 token，編輯模式顯示，且行為與 block id 一致。

7. 修正 decoration 穩定性  
統一採用 `collect -> sort -> add`，避免 out-of-order 導致全域高亮失效。

8. 驗證點擊互動  
檢查點答案進入編輯模式後，不會因第二次點擊跳回閱讀模式。

9. 回歸測試  
跑 `npm test`、`npm run build`，再做手測：單行、填充、多行、錯誤語法、混合語法。

10. 同步文件  
更新 `Manual.md`（只保留最終語法）、`RoadMap.md`（決策與後續）、`Retrospective.md`（本次教訓）。

11. 最終檢查  
確認無多餘語法分支、無暫存 debug、無衝突檔、無 BOM/編碼污染。

12. 版本封版  
整理變更摘要後再 commit（需要時再 push）。
